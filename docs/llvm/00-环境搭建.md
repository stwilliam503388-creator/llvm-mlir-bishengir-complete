# 00 - LLVM 环境搭建

> 目标：配好环境，终端输入 `opt --version` 能看到 LLVM 版本号
> 预估时间：macOS 15 分钟，Linux 10 分钟

---

## macOS（Apple Silicon）

### 安装

```bash
brew install llvm
```

> ⚠️ Homebrew 的 LLVM 是 keg-only — 不会自动加到 PATH。
> 这意味着直接敲 `opt` 会提示 command not found，需要手动加路径。

### 配置 PATH

在 `~/.zshrc` 末尾加上：

```bash
export PATH="/opt/homebrew/opt/llvm/bin:$PATH"
```

然后 `source ~/.zshrc`（或开新终端窗口）。

### 验证

```bash
llvm-config --version   # 应输出: 22.1.6
```

如果你的版本不同（如 18.1.8），没关系，只要 ≥ 18 即可。

```bash
opt --version            # 应输出类似:
                         # LLVM (https://llvm.org/):
                         #   LLVM version 22.1.6
                         #   Optimized build.
```

✅ **验证通过标准**：两条命令都能跑，看到版本号。

---

## Linux（Ubuntu/Debian）

### 方法一：apt 安装（推荐）

```bash
# Ubuntu 24.04+ 默认 LLVM 18
sudo apt install llvm-18-dev clang-18

# 验证
llvm-config-18 --version
```

### 方法二：官方脚本安装最新版

```bash
wget https://apt.llvm.org/llvm.sh
chmod +x llvm.sh
sudo ./llvm.sh 22
```

---

## 常见坑

| 问题 | 现象 | 解决 |
|------|------|------|
| macOS keg-only | `opt: command not found` | 把 `/opt/homebrew/opt/llvm/bin` 加到 PATH |
| cmake 找不到 LLVM | `find_package(LLVM)` 失败 | 加 `-DCMAKE_PREFIX_PATH=$(llvm-config --cmakedir)` |
| Linux 版本太旧 | apt 只有 LLVM 14 | 用官方的 llvm.sh 脚本装新版 |
| 多个 LLVM 版本共存 | llvm-config 指向旧版 | 用 `llvm-config-22` 指定版本 |
