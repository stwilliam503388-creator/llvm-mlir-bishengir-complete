### Dialects 概览

#### 1. `'hacc'` Dialect — HACC 方言

**HACC** = Hardware Abstraction and Compiler Control（硬件抽象与编译器控制）

这个 Dialect 定义了用于指定目标硬件设备和编译器行为的属性（Attributes）和枚举（Enums）。
包含两个主要部分：
- **Attributes**（属性）：描述目标设备名称等信息
- **Enums**（枚举）：定义硬件相关的枚举类型

对应到你的工程中的角色：当需要在 MLIR IR 中标注"这段代码跑在某个具体 NPU 型号上"时，用到的就是 hacc dialect。

#### 2. `'hfusion'` Dialect — HFusion 方言

**HFusion** = Hardware Fusion（硬件融合）

这是**最大的方言之一（43KB / 1660行）**，定义了与算子融合（operator fusion）相关的 MLIR 操作。

包含三个部分：
- **Operations**（操作）：融合相关的 IR 指令（约 1130 行）
- **Attributes**（属性）：融合操作的配置属性（约 290 行）
- **Enums**（枚举）：融合相关的枚举类型（约 240 行）

作用：将多个连续的小算子（如 element-wise add → relu → multiply）合并为单个计算核（kernel），
减少中间结果的访存开销和 kernel 启动次数。

#### 3. `'hivm'` Dialect — HIVM 方言（最大，172KB / 5200行）

**HIVM** = Hardware Independent Virtual Machine（硬件无关的虚拟机）

这是 AscendNPU IR 中**最核心、最庞大**的 Dialect。相当于整个编译器的"骨架"——定义了 NPU 上所有通用操作。

包含三个部分：
- **Operations**（操作）：约 4150 行的操作定义——内存管理、同步、数据搬运、数学运算等
- **Attributes**（属性）：约 630 行的属性定义
- **Enums**（枚举）：约 420 行的枚举定义

典型操作包括：
  - `hivm.add_ffts_to_syncblocksetop` — 向同步块集合添加 FFTs（自由帧标记）
  - `hivm.inject_sync` — 注入同步指令
  - `hivm.alloc_workspace` — 分配工作空间内存
  - `hivm.decompose_op` — 操作拆分（将复合操作拆为基本操作）
  - `hivm.flatten_ops` — 操作展平

可以理解为：HIVM 方言定义了 NPU 编译器在"中间阶段"看到的所有指令的 IR 表示。

#### 4. `'symbol'` Dialect — Symbol 方言

定义符号形状（symbolic shape）相关的操作。

**Operations**（操作）：
  - `symbol.bind_symbolic_shape` — 将张量编码绑定到符号形状
  - `symbol.get_shape_of` — 获取符号形状

作用：在处理动态形状（dynamic shapes）时，用符号变量代替具体数值来表示张量维度。
这在大模型推理中特别重要——batch size 和 sequence length 通常是动态的。

#### 5. `'mathExt'` Dialect — Math 扩展方言

标准 MLIR `math` dialect 的扩展，补充分数支持的数学运算：

**Operations**（操作）：
  - `mathExt.ilogb` — 求浮点数的指数分量（floor(log2(abs(x)))）
  - 其他扩展数学运算

每个操作支持：`AlwaysSpeculatableImplTrait`（可推测执行）、`Elementwise`（逐元素）、
`SameOperandsAndResultType`（操作数与结果同一类型）等特性。

#### 6. `'memref_ext'` Dialect — MemRef 扩展方言

标准 MLIR `memref` dialect 的扩展，为 Ascend NPU 增加的工作区内存管理操作：

**Operations**（操作）：
  - `memref_ext.alloc_workspace` — 分配工作区内存（workspace memory），支持动态大小和偏移
  - 其他 NPU 特有的内存操作

作用：NPU 上需要有专门的"工作区"（workspace）内存用于中间结果，这个 dialect 提供了管理这些内存的 IR 指令。

#### 7. `'scope'` Dialect — Scope 方言

**Operations**（操作）：
  - `scope.scope` — 创建一个作用域（scope）
  - `scope.yield` — 从作用域中产生输出值

对应 MLIR 中的 SSA（静态单赋值）的区域（region）管理。
作用：在 IR 中创建嵌套的作用域，用于管理局部变量和资源的生命周期。

#### 8. `'annotation'` Dialect — Annotation 方言

**Operations**（操作）：
  - `annotation.mark` — 用键值对属性对 IR 值进行标注

参数：`keys`（键列表）、`values`（值列表）
作用：在 IR 中的某个值上附加元数据（metadata），供后续 pass 消费。
不会改变语义，只是打标签。

## 三、整体架构关系

### Dialect 与 Pass 的对应关系

```
  Dialect（方言/IR定义层）          Pass（变换/优化层）
  ──────────────────────           ──────────────────────
  'hivm'  HIVM 方言 (172KB)   →   HIVMPasses (60个Pass)
  'hfusion' HFusion 方言 (43KB) →   HFusionPasses (3个Pass)
  'symbol' 符号方言 (3.1KB)    →   SymbolPasses (5个Pass)
  'scope' 作用域方言 (1.5KB)   →   ScopePasses (2个Pass)
  'hacc' 硬件抽象方言 (8.9KB)  →   HACCPasses (2个Pass)
  'mathExt' 数学扩展示例 (5.2KB) —     (无对应Pass)
  'memref_ext' 内存扩展 (3.3KB) —     (无对应Pass)
  'annotation' 标注方言 (1.5KB) →   AnnotationPasses (1个Pass)
```

### 编译流程中的角色

```
          前端 IR                    中间 IR                    后端
  ┌──────────────┐    ┌─────────────────────────────┐    ┌──────────┐
  │  Triton IR   │    │  AscendNPU IR (this layer)  │    │  NPU 指令│
  │  ONNX IR     │ →  │  ┌───────────────────────┐  │ →  │          │
  │  TileLang IR │    │  │  HIVM + HFusion + ... │  │    │          │
  └──────────────┘    │  │  + ~70 个 Passes       │  │    └──────────┘
                       │  └───────────────────────┘  │
                       └─────────────────────────────┘
                                ↑
                        这就是 developer_guide 覆盖的内容
```

### 关键术语翻译对照表

| 英文 | 中文 | 解释 |
|------|------|------|
| Dialect | 方言 | MLIR 中的一种 IR 语言变体，定义了特定领域的操作和类型 |
| Pass | 变换/遍 | 对 IR 进行一次遍历并做某种变换或分析的算法 |
| Operation | 操作 | IR 中的一条指令（如加法、加载、存储） |
| Attribute | 属性 | 附加在操作上的元数据（如数据类型、对齐方式） |
| Enum | 枚举 | 一组命名常量，用于表示有限的选项集合 |
| Lowering | 降级 | 将高层次 IR 翻译为低层次 IR |
| Sync/Synchronization | 同步 | 协调多个执行单元的计算进度 |
| Fusion | 融合 | 将多个操作合并为一个计算核 |
| Scope | 作用域 | IR 中的变量/资源可见范围 |
| Workspace | 工作区 | NPU 上的临时内存区域 |
| Symbolic Shape | 符号形状 | 用符号变量表示的张量维度（非具体数字） |
| Tile/Tiling | 分块 | 将大张量切分为小块处理 |
| Schedule | 调度 | 决定操作执行顺序的策略 |
| Cube Unit | Cube 单元 | 昇腾 NPU 中做矩阵乘的专用硬件单元 |
| Vector Unit | Vector 单元 | 昇腾 NPU 中做逐元素运算的硬件单元 |
| FFTs | 自由帧标记 | Free Frame Tags，用于跟踪和管理内存帧的机制 |

### 对开发者的实际意义

1. **如果要添加一个新的 NPU 操作**（如自定义算子）：在 `td/` 文件添加定义 → `mlir-tblgen`
   自动生成 dialect 文档和 C++ 代码
2. **如果要添加一个新的优化**：在对应的 passes 目录添加 pass 实现 → 更新相关文件
3. **最大文件是 HIVM Dialect**（172KB / 5200 行），说明它是最核心的底层 IR
4. **HFusion 是一个完整子系统**（Dialect 43KB + Passes 10KB + AutoSchedule/AutoSync等 features）
5. **所有文件都标记为勿手动编辑**——改动需要修改 TableGen `.td` 源文件，重新生成
