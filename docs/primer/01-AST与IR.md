## 2.1 代码是怎么被"理解"的

编译器拿到你的代码，第一件事——**从字符串里提取"意思"**。

```
"a + b * 3"
```

在计算机眼里就是 `['a', ' ', '+', ' ', 'b', ' ', '*', ' ', '3']`——一串字符，毫无意义。

编译器要做两件事：
1. **解析**：认出这是一个表达式，而不是乱打的字符
2. **结构化**：看出 `*` 比 `+` 优先级高，所以先算 `b * 3`，再加 `a`

这就是 AST 要做的事。

---

## 2.2 AST——语法树，代码的"骨架"

AST（Abstract Syntax Tree，抽象语法树）把代码表示成一棵树：

```text
输入: a + b * 3

        ┌─── + ───┐
        a      ┌── * ───┐
               b        3
```

**叶子节点**是变量和常量（`a`、`b`、`3`）。
**根节点**是运算符（`+`、`*`）。**树的结构**体现了运算优先级——`*` 在更下层，先算。

这就是 `projects/toy-mini/toymini.cpp` 做的事：解析你的代码，建成一棵树。

> ✅ **检查自己**：`a * b + 3` 的 AST 长什么样？
> → 根是 `+`，左子树是 `*`（a, b），右子树是 `3`。

## 2.3 IR——编译器的"翻译笔记"

AST 告诉编译器"代码的意思"，但计算机还不能执行它。IR（Intermediate Representation，中间表示）把树变成**一条一条可以按顺序执行的指令**：

```text
AST:          a + b * 3
              ↓ 翻译
IR:     %1 = mul b, 3      ← 先算乘法
        %2 = add a, %1     ← 再加（用乘法的结果）
```

为什么不用 AST 直接执行？因为**树适合给人理解，指令流适合给机器执行**。就像做饭菜谱：

| 形式 | 例子 | 适合谁 |
|------|------|--------|
| AST（树） | "把肉和菜一起炒" | 人看 |
| IR（指令流） | 1.洗菜 2.切肉 3.热锅 4.倒油 5.炒 6.出锅 | 机器执行 |

本项目中，每个 `.mlir` 文件里写的就是 IR：

```mlir
%sum = arith.addf %a, %b : f16    ← "把 %a 和 %b 加起来，存到 %sum"
```

> 💡 **SSA（静态单赋值）**是 IR 的一个重要特性——每个变量只赋值一次。它是进阶内容，在初学阶段你只需知道：看到 `%0, %1, %2` 就是 SSA 变量，每个只能在代码里出现一次（赋值一次，用很多次是可以的）。
