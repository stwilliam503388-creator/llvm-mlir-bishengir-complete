# 技术术语速查手册

> 面向 AI 工程师的编译器术语速查，按主题分组。

---

## 一、LLVM IR 基础

### SSA (Static Single Assignment，静态单赋值)
**一句话**: 每个变量只赋值一次。
**类比**: 就像身份证号——一个人只有一个号，永远不会改。
**为什么重要**: 一眼能看出数据依赖关系——`%2` 用了 `%1`，那 `%1` 一定在 `%2` 之前。
**在项目中的位置**: LLVM IR 的核心设计，`%0`, `%1`, `%2` 就是 SSA 变量。

### GEP (GetElementPtr，获取元素指针)
**一句话**: 计算数组/结构体里某个元素的地址，但不访问它。
**类比**: 外卖小哥看地图找地址——只看地址(指针)，不进去(不读数据)。
**为什么重要**: LLVM 类型系统的核心，理解 GEP 才算懂 LLVM 指针。

### phi (φ) 节点
**一句话**: 当两条路汇合时，选"刚才走的是哪条路"。
**类比**: 两个队伍汇合成一个队伍——每个人要知道自己是从左队还是右队来的。
**为什么重要**: SSA 的"每个变量只赋值一次"遇到分支时，需要用 φ 合并。

### IR (Intermediate Representation，中间表示)
**一句话**: 编译器内部的"通用语言"——人和机器都不直接读写，但编译器用它做优化。
**类比**: 中文(源码)→英文笔记(IR)→日文(机器码)。英文笔记不是给谁看的，是翻译过程中的中间产物。
**为什么重要**: 有了 IR，换硬件只需换后端(日文→法文)，不需要重新做前端。

## 二、MLIR 核心概念

### Dialect (方言)
**一句话**: 一组专为某个领域设计的操作集合。MLIR 有多个 dialect，每个管一摊事。
**类比**: 三个人接力翻译英→日→中，每个人只翻自己最擅长的段落。
**为什么重要**: 传统编译器用 1 个 IR 干所有事，MLIR 用 N 个 IR 各管一段，更灵活。

### Operation (操作)
**一句话**: dialect 里的"动词"——`toy.add` 表示加法，`linalg.matmul` 表示矩阵乘。
**类比**: 每个方言有自己的词汇表，词汇就是 operation。

### Region (区域)
**一句话**: operation 的作用范围。一个 operation 内部可以嵌套另一个操作序列。
**类比**: 函数的花括号 `{}` 定义了变量的作用范围。Region 就是 MLIR 版的 `{}`。

### Pass (遍/趟)
**一句话**: 遍历一次 IR 并做一次修改。
**两种 Pass**:
- **分析 Pass**: 只看不改，统计信息（像质检员数产品数量）。
- **转换 Pass**: 边看边改，做优化（像质检员顺手修毛刺）。
**在项目中的位置**: `BishengirOpCounter.cpp` (分析)、`BishengirPeelTranspose.cpp` (转换)。

### Lowering (降级)
**一句话**: 把"高级但难执行"的 IR 变成"低级但易执行"的 IR。
**类比**: "做满汉全席"(高层语义)→"先买菜再切菜再炒菜"(低层指令)。
**为什么重要**: 中间层可以做中间层优化(比如循环融合)，一步到位会丢失优化机会。
**在项目中的位置**: `bishengir-demo` 的三阶段降级。

### mlir-opt
**一句话**: MLIR 的命令行瑞士军刀——加载 IR、跑 Pass、输出结果。
**类比**: gcc/clang 的 `-O1` `-O2` 优化选项，但针对 MLIR。
**用法**: `mlir-opt --convert-linalg-to-affine-loops input.mlir`

### mlir-tblgen
**一句话**: 从 `.td` 文件自动生成 C++ 代码的工具。
**类比**: 写一个 JSON 配置，自动生成对应的 CRUD 代码——省去手写样板代码。
**用途**: 定义 dialect、operation、类型时使用，一次定义到处用。

### TableGen (ODS — Op Definition Spec)
**一句话**: 定义 dialect 和 operation 的配置文件格式。
**在项目中的位置**: `StandaloneOps.td`、`bishengir/HFusionOps.td`。
**示例**:
```tablegen
def AddOp : Standalone_Op<"add"> {
  let summary = "加法操作";
  let arguments = (ins AnyType:$lhs, AnyType:$rhs);
  let results = (outs AnyType:$result);
}
```

## 三、MLIR Dialect (按学习顺序)

### linalg (Linear Algebra Dialect)
**一句话**: 描述矩阵/张量运算的高级 dialect。
**核心操作**: `linalg.generic` (通用)、`linalg.matmul` (矩阵乘)、`linalg.conv` (卷积)。
**为什么重要**: bishengir 的输入就是 linalg dialect。它是 AI 编译器的"入口 dialect"。
**在项目中的位置**: `bishengir-demo` 的测试用例全都是 linalg dialect。

### affine (仿射 Dialect)
**一句话**: 描述循环和数组访问的 dialect，比 linalg 低一级。
**核心操作**: `affine.for` (循环)、`affine.load` (读)、`affine.store` (写)。
**类比**: linalg = "我要算矩阵乘法"(高级)，affine = "for i=0..4, for j=0..4"(中级)。
**在项目中的位置**: bishengir-demo 的 Stage1 输出。

### scf (Structured Control Flow Dialect)
**一句话**: 结构化控制流——if-else、while 循环、for 循环。
**类比**: Python 的 `if/for/while` 而不是 goto。
**在项目中的位置**: bishengir-demo 的 Stage2 降级经过。

### func / cf (Function / Control Flow Dialect)
**一句话**: func 表示函数，cf 表示控制流(跳转/分支)。

### arith (Arithmetic Dialect)
**一句话**: 加减乘除等算术操作。
**核心操作**: `arith.addf`、`arith.mulf`、`arith.cmpf`(比较)、`arith.select`(选择)。
**在项目中的位置**: bishengir-demo 几乎每个用例都用。

### math (Math Dialect)
**一句话**: 数学函数：exp、tanh、sqrt 等。
**核心操作**: `math.exp`(指数)、`math.tanh`(双曲正切)、`math.sqrt`(开平方)。
**在项目中的位置**: softmax、sigmoid、tanh 等测试用例。

### memref (Memory Reference Dialect)
**一句话**: 描述内存缓冲区的类型和操作。
**核心操作**: `memref.alloc`(分配)、`memref.dealloc`(释放)、`memref.load`(读)、`memref.store`(写)。
**类比**: C 语言的 `malloc/free` + 指针的读写。
**在项目中的位置**: 大部分测试用例的输入/输出都是 `memref` 类型。

### tensor (Tensor Dialect)
**一句话**: 描述不可变的张量(与 memref 相对).
**核心类型**: `tensor<4x4xf32>` (4行4列的f32张量)。
**与 memref 的区别**: tensor 是值语义(不可变，每次操作创建新tensor)，memref 是引用语义(可原地修改)。

## 四、bisenghir / AscendNPU-IR 专用

### hfusion (HFusion Dialect)
**一句话**: 华为 Ascend NPU 的"算子融合" dialect——把多个操作合并成一个。
**核心操作**: `hfusion.elemwise_binary` (逐元素二元)、`hfusion.cube_matmul` (矩阵乘法)。
**类比**: linalg 是"我要做加法"，hfusion 是"我要用 NPU 的特定单元做加法"。
**在项目中的位置**: bishengir 三阶段降级的第二阶段。

### hivm (HIVM Dialect)
**一句话**: 华为 Ascend NPU 的"指令级" dialect——最接近硬件的 IR。
**核心操作**: `hivm.vadd` (向量加法)、`hivm.mmul` (矩阵乘法)、`hivm.load`/`hivm.store`。
**类比**: 汇编指令，一条指令对应一个硬件操作。
**在项目中的位置**: bishengir 三阶段降级的最终输出。

### Cube 单元
**一句话**: Ascend NPU 上专门做矩阵乘法的硬件单元。
**类比**: CPU 有 ALU(算术单元)，NPU 有 Cube(矩阵乘单元)。一个 Cube 指令=几百条 CPU 指令。
**为什么重要**: 为什么 bishengir 能把 74 行 LLVM IR 保持为 1 行——就是因为 Cube 单元在硬件层面直接执行矩阵乘法。

### CANN (Compute Architecture for Neural Networks)
**一句话**: 华为昇腾的 SDK，负责把编译后的代码加载到 NPU 并执行。
**在项目中的位置**: bishengir 输出 HIVM → CANN → NPU 执行。

### bishengir-opt
**一句话**: bishengir 的命令行工具，类似 mlir-opt 但针对 Ascend NPU。
**用法**: `bishengir-opt --convert-linalg-to-hfusion input.mlir`

### ConversionTarget
**一句话**: 告诉编译器"哪些操作是合法的最终输出"，不属于目标的操作都要被转换。
**类比**: 装修时，你告诉装修队"我只要现代风格"，所有欧式/中式设计都要改。
**在项目中的位置**: 每个 bishengir Conversion Pass 的开头都有 `ConversionTarget` 设置。

### OpRewritePattern
**一句话**: 定义"把一种操作转换成另一种操作"的规则。
**类比**: "见到'五毛'就换成'0.5元'"——一条替换规则。
**在项目中的位置**: `BishengirPeelTranspose.cpp`、bishengir 的 `LinalgGenericToHFusion`。

## 五、深度学习概念

### 残差连接 (Residual Connection / Shortcut)
**一句话**: y = x + F(x)。把输入 x 直接加到输出上。
**类比**: 写文章时保留原稿——改完一段后把原稿和新稿叠在一起。
**为什么重要**: 解决深层网络的梯度消失问题。ResNet/Transformer/GPT 全靠这个才能训练。

### 激活函数 (Activation Function)
**一句话**: 给神经网络增加"非线性能力"的函数。如果没有它，再多层网络也等于一层。
**常见类型**:
- **ReLU**: `max(0, x)`，最快但太"硬"。
- **GELU**: 平滑版 ReLU，BERT 用。
- **SiLU**: x × sigmoid(x)，LLaMA 用。
- **Sigmoid**: 压缩到 0~1，RNN 门控用。
- **Tanh**: 压缩到 -1~1，RNN 用。

### 算子融合 (Kernel Fusion)
**一句话**: 把连续两个 kernel 合并为一个，减少内存读写。
**类比**: 边切菜边炒菜，而不是等所有菜切完再炒。
**为什么重要**: 性能提升 1.3-2 倍，深度学习编译器(TVM/XLA/Triton)的核心优化。

### 批归一化 (Batch Normalization, BN)
**一句话**: 对一批数据做标准化(均值=0, 方差=1)，然后学习缩放和平移。
**类比**: 让所有人脱鞋站直再量身高，然后 AI 自己决定要不要穿高跟鞋。
**用在哪**: ResNet 每层卷积后 (Conv→BN→ReLU)。LLM 不用(不适合变长文本)。

### 层归一化 (Layer Normalization, LN)
**一句话**: 对每个 token 自己做标准化(不依赖 batch 内其他样本)。
**类比**: 每个人量自己身高，不用和别人比。
**为什么重要**: 适合变长序列(NLP/LLM)，GPT/BERT/LLaMA 每层都有。

### Attention (注意力机制)
**一句话**: 让模型在生成每个词时，知道"该看输入的哪些位置"。
**类比**: 翻译时: "I love you" → 翻译"我"时看"I"，翻译"爱"时看"love"。
**核心公式**: `softmax(QK^T / √d) × V`

### Softmax
**一句话**: 把一堆数变成"概率分布"——所有数加起来等于 1，每个数在 0~1 之间。
**公式**: softmax(x_i) = e^{x_i} / Σe^{x_j}
**类比**: 几个人的考试成绩 [90, 80, 70] → 排名占比 [0.45, 0.35, 0.20]。

### 归约 (Reduction)
**一句话**: 把多个数变成一个数：求和、求最大值、求平均值。
**类比**: 全班考试成绩 → 算平均分(归约)。
**在MLIR中**: `iterator_types = ["reduction"]` 表示这个维度要被"归约掉"。

### 广播 (Broadcasting)
**一句话**: 把小形状自动扩展到和大形状一样，然后做运算。
**类比**: 广播体操——一个人在台上做动作，全场几千人跟着做(一个数扩展到所有位置)。

### 池化 (Pooling)
**一句话**: 把图片缩小(下采样)，同时保留重要信息。
**两种**: Max Pool(取最大值)、Avg Pool(取平均)。
**类比**: 给高清照片做缩略图——每 4 个像素合成 1 个像素。

### 梯度裁剪 (Gradient Clipping)
**一句话**: 训练时限制梯度的大小，防止一步更新太多(梯度爆炸)。
**类比**: 高速公路限速——不能让车开到 200km/h。

### Dropout (随机丢弃)
**一句话**: 训练时随机让一部分神经元"翘课"(输出置0)，防止过拟合。
**类比**: 每次上课随机抽人回答问题——逼所有人都得学会，不能依赖个别"学霸"。

## 六、工程与工具

### linalg.generic
**一句话**: linalg dialect 最通用的操作——自己定义循环体做什么。
**示例**: 想要做加法就写 `arith.addf`，想要做乘法就写 `arith.mulf`。
**在项目中的位置**: bishengir-demo 大部分用例都用这个。

### affine.for
**一句话**: 显式写循环。比 linalg.generic 更低级，但更灵活。
**在项目中的位置**: max_pool/avg_pool 等不能用 linalg.generic 表达的操作。

### memref vs tensor
**一句话**: memref 可原地修改，tensor 不可变(每次操作返回新 tensor)。
**类比**: memref = 黑板(写了可以擦掉重写)，tensor = 便利贴(每次写新的一张)。

### linalg.matmul
**一句话**: 命名操作，直接表达矩阵乘法，不需要写循环。
**在项目中的位置**: `03_advanced/01_matmul.mlir`。
**为什么重要**: 1 行 → 74 行 LLVM IR，展示了"高级语义保留"的价值。

### LinalgToHFusion (bishengir Pass1)
**一句话**: 把 linalg 操作转成 hfusion 操作。
**源码位置**: `bishengir/lib/Conversion/LinalgToHFusion/`

### ArithToHFusion (bishengir Pass2)
**一句话**: 把 arith 操作转成 hfusion 操作。
**源码位置**: `bishengir/lib/Conversion/ArithToHFusion/`

### HFusionToHIVM (bishengir Pass3)
**一句话**: 把 hfusion 操作转成 hivm 指令(NPU 指令)。
**源码位置**: `bishengir/lib/Conversion/HFusionToHIVM/`
