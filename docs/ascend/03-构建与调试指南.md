# 03 — 构建与调试指南

> 目标：在自己机器上构建 ascendnpu-ir 并运行测试
> 前置：[02 — 一个 Ascend Pass 详解](./02-一个Ascend-Pass详解.md)
> 预估时间：30 分钟（不含 LLVM 编译）

## 1. 前置依赖

| 依赖 | 安装 |
|------|------|
| LLVM/MLIR ≥ 18 | `brew install llvm` |
| CMake ≥ 3.20 | `brew install cmake` |
| Ninja | `brew install ninja`（加速） |

```bash
./setup.sh   # 快速检查
```

## 2. 构建

```bash
git clone https://github.com/Ascend/AscendNPU-IR.git
cd AscendNPU-IR

export PATH="/opt/homebrew/opt/llvm/bin:$PATH"

mkdir build && cd build
cmake .. -G Ninja \
  -DCMAKE_BUILD_TYPE=Release \
  -DLLVM_DIR="/opt/homebrew/opt/llvm/lib/cmake/llvm"

ninja -j$(sysctl -n hw.logicalcpu)
```

> 首次构建 10-30 分钟。

## 3. 运行测试

```bash
# 全部测试
ninja check-ascendnpu-ir

# 单个测试
./bin/ascendnpu-ir-opt \
  --pass-pipeline="builtin.module(convert-linalg-to-hfusion)" \
  ../bishengir/test/Conversion/convert-linalg-to-hfusion.mlir
```

## 4. 调试

### 加打印
```cpp
llvm::errs() << "DEBUG: matched " << op->getName() << "\n";
```

### 看每步 IR
```bash
./bin/ascendnpu-ir-opt \
  --mlir-print-ir-after-all \
  --pass-pipeline="convert-linalg-to-hfusion" \
  test.mlir
```

### 逐步 Lowering
```bash
# 只看 linalg → husion 这一步的输出
./bin/ascendnpu-ir-opt \
  --pass-pipeline="builtin.module(convert-linalg-to-hfusion)" \
  input.mlir -o step1.mlir
cat step1.mlir
```

## 5. 常见坑

| 问题 | 解决 |
|------|------|
| cmake 找不到 MLIR | 确认 `LLVM_DIR` 指向正确路径 |
| LLVM 版本不匹配 | `brew install llvm` 确保 ≥ 18 |
| 构建内存不足 | `ninja -j2` 减少并行 |
| 找不到 ascendnpu-ir-opt | 二进制在 `build/bin/` 下 |

## 6. 学习建议

| 阶段 | 做什么 | 时间 |
|------|--------|------|
