## 1. NPU vs GPU vs CPU

| | CPU | GPU | NPU |
|---|-----|-----|-----|
| 核心数 | 几个大核 | 几千个小核 | 专用矩阵单元 |
| 强项 | 复杂逻辑、分支 | 大规模并行 | 矩阵乘法（AI） |
| 类比 | 厨师长 | 100 个帮厨 | 切菜机 |

**NPU 为 AI 推理/训练设计**。核心能力：
1. 矩阵乘法极快 — 神经网络本质是矩阵乘法
2. 数据搬运是瓶颈 — 计算快但内存慢

## 2. Da Vinci 架构

华为 Ascend 芯片采用 Da Vinci（达芬奇）架构：

```
┌─────────────────────────────────┐
│          Da Vinci Core           │
│  ┌──────────┐ ┌──────────┐      │
│  │Cube Unit │ │ Vector   │      │
│  │(矩阵乘法)│ │ Unit     │      │
│  │ 16×16    │ │(向量运算)│      │
│  └──────────┘ └──────────┘      │
│       ↑            ↑            │
│  ┌──────────────────────────┐    │
│  │    L1 Buffer (1MB)       │    │
│  └──────────────────────────┘    │
│            ↑                     │
│  ┌──────────────────────────┐    │
│  │    L2 Cache (8MB)        │    │
│  └──────────────────────────┘    │
│            ↑                     │
│  ┌──────────────────────────┐    │
│  │    HBM (32GB+)           │    │
│  └──────────────────────────┘    │
└─────────────────────────────────┘
```

**厨房类比**：

| 硬件 | 厨房 | 说明 |
|------|------|------|
| Cube Unit | 切菜机 | 矩阵乘法 16×16 |
| Vector Unit | 炒锅 | 向量加减乘除 |
| L1 Buffer | 案板 | 手边，容量小 |
| HBM | 冰箱 | 容量大，要走两步 |

## 3. 编译器要做什么？

```
PyTorch 算子 (Conv2D)
     ↓
linalg.generic (MLIR 高级)
     ↓  ← 编译器后端开始
husion (昇腾融合 IR)
     ↓
hivm (昇腾虚拟指令)
     ↓
Ascend 可执行文件
```

**三个核心挑战**：

| 挑战 | 说明 | 类比 |
|------|------|------|
| 算子融合 | 多个操作合并，减少数据搬运 | 切菜+炒菜一站完成 |
| 内存管理 | 何时 HBM→L1，何时 L1→HBM | 冰箱→案板→炒锅时机 |
| 指令映射 | matmul → Cube Unit 指令 | 菜谱→切菜机操作 |

## 4. 和 CUDA 的对比

| | CUDA (NVIDIA) | Ascend (华为) |
|---|--------------|--------------|
| 编译器 | NVCC → PTX → SASS | MLIR → husion → hivm |
| IR 体系 | 私有 | 开放（MLIR） |
| 可迁移性 | 局限 NVIDIA | MLIR 通用技能 |

> 学 Ascend 编译器的隐藏优势：MLIR 技能可迁移到任何硬件后端。
